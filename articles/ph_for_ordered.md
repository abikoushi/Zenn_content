## このノートについて

目的変数が順序尺度の場合の回帰分析を考えるとき，順序ロジスティック回帰（比例オッズモデルとも呼ばれる）や順序プロビット回帰を用いることができる．それらに比べると見かける機会は少ないが，比例ハザードモデルでも目的変数が順序尺度の場合の回帰を扱うことができる．

## モデルの紹介

順序尺度の確率変数 $Y$ が $1,...,k$ の整数の値をとるとして，$j$ 以上になる確率を次のように表すことを考える．

$$
P(Y \le j) = F(c_j - \eta)
$$

順序ロジスティック回帰は $F(x)$ にロジスティック関数を用いる．このことは次のように考えるとイメージしやすいかもしれない．

適当な連続型の潜在変数 $Z$ があり，

- $Z$ が閾値 $c_1$ を下回ったとき$Y=1$, 
- $Z$ が閾値 $c_1$ 以上 $c_2$ 未満のとき $Y=2$, 
- $Z$ が閾値 $c_{j-1}$ 以上 $c_j$ 未満のとき $Y=j$ （ここでは $j=2, \ldots, k-1$）, 
- $Z$ が閾値 $c_{k}$ 以上のとき $Y=k$

![](/images/ph_for_ordered/IMG_2142.jpg)
*ロジスティック分布を離散化して考える*

そして潜在変数の分布のロケーションを規定するパラメータが $\eta$ である．

これは各観測 $1, 2, \ldots , k$ を区間 $(-\infty, c_1), [c_1, c_2), \ldots , (c_k, \infty)$ の打ち切りデータと扱っているのと同じである．

さて，各観測を区別するために添字 $i$ をつけ，適当な説明変数のベクトル $x_i$ と係数のベクトル $\beta$ を用いて $\eta_i = x_i' \beta$ とする．$Y_i$ の確率関数を書き下すと次の箇条書きのようになる． 

- $P(Y_i = 1) = F(c_k - \eta_i)$
- $P(Y_i = 2) = F(c_2 - \eta_i) - F(c_1 - \eta_i)$
- $P(Y_i = j) = F(c_2 - \eta_i) - F(c_{j-1} - \eta_i)$（ここでは $j=2, \ldots, k-1$）, 
- $P(Y_i=K) = 1-F(c_k)$

順序ロジスティック回帰が累積分布関数に対して回帰構造を持ち，

$$
P(Y \le j) = \mathrm{logit}^{-1}(c_j-\eta_i)
$$

であるのに対し，ハザードに対して回帰構造を持つ次のような形にすると（離散型の）比例ハザードモデルである．

$$
P(Y = k|Y \ge k) =\mathrm{logit}^{-1}(\alpha_j - \eta_i)
$$

このとき $Y_i$ の確率関数を書き下すと次の箇条書きのようになる． 

- $P(Y = 1) = \mathrm{logit}^{-1}(\alpha_1 - \eta_i)$
- $P(Y = 2) = \mathrm{logit}^{-1}(\alpha_2 - \eta_i) ( 1-P(Y=1))$
- $P(Y = j) = \mathrm{logit}^{-1}(\alpha_j - \eta_i)(1-P(Y \le {j-1}))$（ここでは $j=2, \ldots, k-1$）, 
- $P(Y=k) = 1-P(Y \le {k-1})$

順序ロジスティック回帰のパラメータ $c_j$ は $c_1 < \cdots < c_k$ に制約されるが，比例ハザードモデルの $\alpha_j$ に制約は特に必要ないことを注意しておく．


## R による計算例

R の `MASS` パッケージの `polr` 関数は順序ロジスティック回帰や順序プロビット回帰が実装されているので，この例に比例ハザードモデルも追加してみよう．フィッティングには最尤法を用いる．

ヘルプの例に使われている `housing` データは次の項目が記録されている．

- 満足度（Sat）：現在の住居状況に対する世帯主の満足度（High, Medium, Low）
- 影響力（Infl）：物件管理に対する世帯主の認識している影響力の度合い（High, Medium, Low）
- タイプ（Type）：賃貸住宅のタイプ（Tower, Atrium, Apartment, Terrace）
- 接触頻度（Cont）：居住者同士の接触頻度（Low, High）
- 頻度（Freq）：各カテゴリーの居住者数

まず $P(Y_i=j)$ となる確率をそれぞれのモデルで比較し，当てはまりを確認してみる．

![](/images/ph_for_ordered/fit.png)
*x軸は観測値，y軸はモデルによる推定値.PHは比例ハザードモデル*

ぱっと見では五十歩百歩という感じだが尤度では比例ハザードモデルが小さめになっている．

```r
> print(res$value) #比例ハザードモデル
[1] -1741.624
> print(logLik(fit_l)) #順序ロジスティック回帰
'log Lik.' -1739.575 (df=8)
> print(logLik(fit_p)) #順序プロビット回帰
'log Lik.' -1739.844 (df=8)
```

次に回帰係数とワルド法によるその95％信頼区間を確認してみる．これらの係数は，順序ロジスティック回帰の場合は累積分布のオッズ比の対数，比例ハザードモデルの場合はハザードのオッズ比の対数に相当する．比例プロビットモデルの場合はオッズ比のようなわかりやすい解釈はちょっとみつからない．もしなにかあったら教えてもらえると嬉しい．

![](/images/ph_for_ordered/coef.png)
*回帰係数とその95％信頼区間．PHは比例ハザードモデル*

比例ハザードモデルの係数は順序ロジスティックと順序プロビットの中間くらいの値になっている．

しかし，パラメータの数が同じで尤度が小さめということはAICやBICで比較しても相対的に良くないことを意味する，せっかくやってみたのに残念だが……．まあでも気が向いたら・機会があったら比例ハザードも試してみてください．
